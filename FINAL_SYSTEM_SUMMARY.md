## 🎯 Key Points

### ✅ Fixed Naming Convention
- **Upload files**: `road1.mp4`, `road2.mp4`, `road3.mp4`, `road4.mp4`
- **Data files**: `vehicle_data_road1.txt`, `vehicle_data_road2.txt`, etc.
- **URLs**: `/results/road1`, `/results/road2`, etc.
- NO random or user-defined names allowed

### ✅ Single Results Page
- One Results component handles all roads
- Uses URL parameter: `/results/:road`
- Dynamically loads data based on road parameter

### ✅ Fixed Navigation Flow
1. Upload page: Select videos and upload
2. On success: "View Results" button appears
3. Click "View Results" → Goes to `/results/road1`
4. Road buttons allow switching between roads
5. Data updates automatically

### ✅ Data Processing
- Each road is processed independently
- YOLO generates separate vehicle_data file per road
- No data mixing or conflicts between roads

## ⚠️ Important Behaviors

1. **File Naming is Forced**
   - Uploading "traffic-cam-1.mp4" → Saved as "road1.mp4"
   - Original filename is ignored
   - Ensures consistent naming

2. **Data is Road-Specific**
   - Road1 data only in vehicle_data_road1.txt
   - Road2 data only in vehicle_data_road2.txt
   - No mixing or cross-contamination

3. **No Auto-Navigation**
   - Upload doesn't auto-redirect
   - User must click "View Results"
   - Allows multiple uploads without interruption

4. **Single Page Application**
   - Same `/results/:road` page for all roads
   - Loads different data based on URL parameter
   - Efficient and maintainable

## 🎓 System Architecture

```
Frontend (React + Vite)
├── App.jsx (Router)
├── Upload.jsx (Upload & process)
└── Results.jsx (Display per-road data)

Backend (Flask)
├── /upload (POST) → Save road1.mp4, road2.mp4, etc.
├── /data/<road> (GET) → Return vehicle_data_road{N}.txt
└── yolotest.py → Process video, generate data file

Storage
├── uploads/ → road1.mp4, road2.mp4, road3.mp4, road4.mp4
└── backend/ → vehicle_data_road1.txt, vehicle_data_road2.txt, etc.
```

## 🐛 Debugging Tips

### Check if files uploaded correctly
```bash
ls -la backend/uploads/
# Should show: road1.mp4, road2.mp4, road3.mp4, road4.mp4
```

### Check if data files generated
```bash
ls -la backend/ | grep vehicle_data
# Should show: vehicle_data_road1.txt, vehicle_data_road2.txt, etc.
```

### Check backend logs
Look for:
- `✅ road1: Saved as road1.mp4`
- `🎬 Triggering YOLO for uploads/road1.mp4`
- `✅ YOLO processing completed`

### Check frontend console
Open DevTools (F12):
- Network tab: Check `/data/road1` request
- Console: Check for fetch errors
- Check URL in address bar: `/results/road1`
