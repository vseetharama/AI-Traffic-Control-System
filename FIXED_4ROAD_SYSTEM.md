## System Flow

### Upload Process
```
1. User selects videos for roads
2. Clicks "Process Traffic Data"
3. FormData sent to backend:
   {road1: file, road2: file, road3: file, road4: file}

4. Backend processes each road:
   - Saves as: road1.mp4, road2.mp4, road3.mp4, road4.mp4
   - Runs YOLO for each
   - Creates: vehicle_data_road1.txt, vehicle_data_road2.txt, etc.

5. Success message shown
6. "View Results" button appears

7. User clicks "View Results"
   → Navigates to /results/road1
```

### Results Display
```
1. URL: /results/road1
2. Frontend fetches from: http://localhost:5000/data/road1
3. Backend returns: vehicle_data_road1.txt contents as JSON
4. Shows: Statistics, table, graph for road1

5. User clicks Road2 button
6. URL changes to: /results/road2
7. Fetches from: http://localhost:5000/data/road2
8. Shows: road2's unique data
```

## Troubleshooting

### Issue: Buttons don't navigate
**Solution**: Ensure React Router is working. Check console for errors.

### Issue: "No data available" for all roads
**Solution**: 
1. Check if videos were uploaded
2. Check if YOLO processing completed
3. Look for `vehicle_data_road1.txt` files in backend folder

### Issue: Wrong data displaying
**Solution**: 
- Verify file naming is `vehicle_data_road1.txt`, not something else
- Check backend logs for YOLO errors
- Clear cache and refresh

### Issue: Files not uploading
**Solution**:
- Check backend is running on port 5000
- Check CORS is enabled
- Check file size doesn't exceed 500MB limit
